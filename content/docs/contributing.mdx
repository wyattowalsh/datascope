## Getting Started

### Fork & Clone
```bash
git clone https://github.com/YOUR_USERNAME/datascope.git
cd datascope
npm install
```

### Development
```bash
npm run dev              # Start dev server (localhost:5000)
npm run build            # Build for production
npm run preview          # Preview production build
```

## Making Changes

### Create a Branch
```bash
git checkout -b feature/add-xml-support
git checkout -b fix/graph-memory-leak
```

## Testing Checklist

- [ ] Works in Chrome, Firefox, Safari
- [ ] Responsive on mobile (<768px)
- [ ] Light and dark themes work
- [ ] No console errors
- [ ] Performance acceptable

## Submitting Changes

1. **Update your branch:**
   ```bash
   git fetch upstream
   git rebase upstream/main
   ```

2. **Push to your fork:**
   ```bash
   git push origin feature/your-feature
   ```

3. **Create Pull Request** on GitHub
   - Use the PR template
   - Link related issues
   - Add screenshots for UI changes

## Commit Messages

Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add XML data format support
fix: resolve graph memory leak
docs: update API documentation
perf: optimize tree rendering
```

**Types:** `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`
