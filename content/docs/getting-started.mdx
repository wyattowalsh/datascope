# Getting Started

Get up and running with Data Scope in **under 5 minutes**. This guide covers loading, parsing, and visualizing your data.

## üåê Online (No Installation)

The fastest way to use DataScope:

### Step 1: Visit the Live App
Go to **[datascope.w4w.dev](https://datascope.w4w.dev)** - no sign-up required!

### Step 2: Load Your Data
Choose one of three methods:

**Paste**: Copy and paste data directly into the input area
- Best for: Quick tests, small files (< 1MB)
- All formats supported
- Auto-detection enabled

**Upload**: Drag & drop or click to browse
- Best for: Large files (> 1MB)
- Supports: `.json`, `.yaml`, `.yml`, `.jsonl`, `.csv`, `.json5`
- Progress feedback for large files

**URL**: Load from a public URL
- Best for: APIs, remote datasets
- Enter any URL returning text
- 30-second timeout

### Step 3: Start Exploring
DataScope auto-detects format and parses immediately:
- View data in **Tree View** (default)
- Switch to **2D Graph** or **3D Graph**
- Run analytics and quality checks
- Execute JSONPath queries
- Export in any format

> **Privacy Note**: Your data never leaves your browser. All processing is 100% client-side.

## üíª Local Development

To run DataScope locally:

### Prerequisites
- **Node.js**: v18.0.0 or higher
- **npm**: v9.0.0 or higher
- **Modern browser**: Chrome 90+, Firefox 88+, Safari 14+

### Installation Steps

```bash
# 1. Clone repository
git clone https://github.com/wyattowalsh/datascope.git
cd datascope

# 2. Install dependencies
npm install

# 3. Start dev server
npm run dev

# 4. Open browser
# Navigate to http://localhost:5000
```

The dev server includes hot module replacement (HMR) - changes reflect instantly!

### Build for Production

```bash
# Build optimized version
npm run build

# Preview production build
npm run preview
```

## üìù Your First Analysis

Let's walk through a complete example:

### 1. Load Example Data
Click **Examples** ‚Üí **JSON Example** to load sample user data.

### 2. Explore Tree View
The tree view shows your data structure:
- **Expand/Collapse**: Click arrows to navigate
- **Syntax Highlighting**: Colors indicate data types
- **Path Selection**: Click nodes to select paths
- **Copy Values**: Click copy icon on any node

### 3. Switch to Graph View
Click **Graph** tab to see visual relationships:
- **Nodes**: Each data element
- **Links**: Relationships between elements
- **Layouts**: Try Force, Tree, Radial, or Grid
- **Zoom**: Scroll to zoom, drag to pan
- **Select**: Click nodes to highlight

### 4. Run Analytics
Scroll down to view automatic analytics:
- **Statistics**: Total keys, max depth, type distribution
- **Quality Score**: 0-100 rating with issues flagged
- **Graph Metrics**: Centrality, clustering, density
- **Performance**: Parse time, data size, node count

### 5. Try JSONPath Queries
Use the Query Panel to filter data:

```jsonpath
# Find all users
$.users[*]

# Users over 30
$.users[?(@.age > 30)]

# All email addresses
$..email
```

Results appear instantly with live preview!

### 6. Export Results
Click **Export** and choose format:
- JSON (formatted or minified)
- YAML
- CSV
- JSONL
- TypeScript interfaces
- Plain text

## üìä Supported Formats

### JSON (JavaScript Object Notation)
```json
{
  "name": "DataScope",
  "version": "2.7.0",
  "features": ["parsing", "visualization", "analytics"]
}
```

### YAML (YAML Ain't Markup Language)
```yaml
name: DataScope
version: 2.7.0
features:
  - parsing
  - visualization
  - analytics
```

### JSONL (JSON Lines)
```jsonl
{"id": 1, "name": "Alice"}
{"id": 2, "name": "Bob"}
```

### CSV (Comma-Separated Values)
```csv
id,name,age
1,Alice,30
2,Bob,25
```

### JSON5 (JSON for Humans)
```json5
{
  // Comments allowed
  name: 'DataScope',  // Unquoted keys
  version: '2.7.0',
  trailing: 'comma',  // OK
}
```

## üí° Tips for Best Results

### Performance Tips
- For files > 10MB, use file upload (not paste)
- Virtualization activates automatically for 1000+ nodes
- Use Advanced Search to filter before expanding large trees
- Canvas mode auto-enables for graphs with 5000+ nodes

### Format Tips
- **JSON**: Must be valid (no trailing commas, proper quotes)
- **YAML**: Use spaces for indentation (not tabs)
- **JSONL**: One JSON object per line, no commas between
- **CSV**: First row must be headers, consistent columns

## üêõ Troubleshooting

### Parse Errors
**Problem**: "Failed to parse" error

**Solutions**:
- Check for syntax errors
- Verify format matches selected tab
- Look for special characters or encoding issues
- Try pasting a smaller sample first

### Performance Issues
**Problem**: App feels slow

**Solutions**:
- Use file upload for large datasets
- Wait for Web Worker parsing
- Close other browser tabs
- Clear browser cache and reload

### Visualization Not Showing
**Problem**: Graph or tree is blank

**Solutions**:
- Verify data parsed successfully
- Try refreshing the page
- Switch between visualization modes
- Check browser console for errors

## Understanding the Interface

### Main Input Area
- Large textarea for pasting or editing data
- Format detection badge showing the detected format
- Tools dropdown for formatting and minifying
- Parse button to process the data

### Visualization Panel
After parsing, you'll see three view modes:
- **Tree** - Hierarchical structure with expand/collapse
- **2D Graph** - Interactive force-directed graph
- **3D Graph** - 3D spatial visualization

### Right Sidebar
Multiple analysis panels:
- Quick Actions - Common operations
- Advanced Search - Regex and type filters  
- Smart Suggestions - Context-aware recommendations
- Quick View - Selected node details
- Favorites - Save important paths
- Data Validator - Quality checks
- Performance Metrics
- Statistics Panel
- Graph Analytics

## Basic Workflow

5. **Analyze** - Review statistics and analytics
6. **Transform** - Apply transformations if needed
7. **Export** - Download in various formats
